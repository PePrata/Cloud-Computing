# Cloud-Computing â€” Microservices Shop

A small distributed shop system (API gateway + user/product/order microservices)
deployed to AWS with Terraform and Ansible, with an automated multi-region
disaster-recovery (DR) capability layered on top.

## What this is

- **4 Spring Boot services** (Java 21): `api-gateway`, `user-service`,
  `product-service`, `order-service`. The gateway reverse-proxies to the other
  three; `order-service` validates against `user-service`/`product-service`
  synchronously (OpenFeign) and publishes events to SQS for `product-service`
  to consume asynchronously (stock updates).
- **Infrastructure as code**: Terraform modules (`vpc`, `security`, `db`,
  `messaging`, `ecr`, `compute`, `secrets`, `dr-controller`) composed into
  three root environments: `dev` (primary, `us-east-1`), `dr` (standby,
  `eu-west-1`), and `dr-controller` (the Route 53 failover wiring that spans
  both).
- **Configuration management**: Ansible pulls DB credentials from SSM
  Parameter Store on the target host itself and renders/starts the
  `docker-compose` stack for whichever region it targets.
- **CI/CD**: GitHub Actions, OIDC-authenticated (no long-lived AWS keys),
  with `terraform plan` on every pull request for both regions, gated
  `terraform apply` on merge to `main`, and a manually-triggered failover
  drill workflow.

See `docs/architecture.md` for the full system design and diagrams, and
`docs/dr.md` for the disaster-recovery runbook.

```
  inventory/primary/       Generated by terraform/environments/dev

  deploy.yml               terraform apply (both regions) + build/push images + Ansible deploy, on merge to main

docs/                      Full technical documentation (see below)
```

## How to deploy

1. One-time bootstrap (Terraform state bucket/lock table, OIDC role, GitHub
   secrets, EC2 key pairs in both regions): see `docs/setup.md`.
2. Push to a feature branch and open a PR â€” CI runs tests and
   `terraform plan` for both `dev` and `dr`.
3. Merge to `main` â€” the CD pipeline applies Terraform to the primary,
   then the standby, then the DR controller, then builds/pushes images and
   runs Ansible against both regions. Full step-by-step in
   `docs/deployment.md`.

## How to run locally

Each service can run standalone with its local (H2) profile:

```bash
cd services/<service-name>
mvn spring-boot:run
```

Or the whole stack via the same `docker-compose.yml` Ansible renders in
production â€” see `docs/setup.md` for local prerequisites.
